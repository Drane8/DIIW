UT1. Planificación interfaces web

Etiquetas semnánticas HTML5

<header>

Elemento <header> representa un grupo de elementos introductorios o de navegación de una sección o página. Si su elemento padre es el body , se le considera la cabecera de la página entera. Generalmente está cerca de su elemento padre.


<hgroup>

La etiqueta <hgroup> se usa para agrupar un conjunto de elementos de encabezado, <h1>– <h6>. El uso más típico de esta etiqueta es para agrupar el título de la página con su eslogan.


<main>

El elemento <main> representa el contenido principal del body de un documento. No es una sección del contenido y no tiene efecto en el documento outline. El contenido principal incluye contenido único, por lo tanto excluye contenido que se repite a través de la página como links de navegación, copyright, logos, banners y formularios de búsqueda,... Los usuarios que utilizan el teclado como método de navegación al contenido están enfrentados respecto a la manera de acceder al contenido principal o elemento main. El elemento main debe ser único y no debe descender o ser hijo de elementos como article, aside, footer, header or nav.


<section>

La etiqueta <section> se usa para marcar una sección genérica de un documento, con un encabezado. Sólo se debería usar <section> para contenido al que se podría poner un encabezado y que no sea susceptible de ir marcado con article, aside o nav. Por ejemplo un capítulo de un libropuede ser una sección y debe ir acompañado de h1-h6 para estructurar mejor toda la página creando jerarquías del contenido, algo muy favorable para la accesibilidad Web y el buen posicionamiento web.


<article>

La etiqueta <article> se usa para marcar contenido independiente que tendría sentido fuera del contexto de la página actual y que podría sindicarse, por ejemplo: una noticia, un artículo en un blog o un comentario. Normalmente un <article> podrá además tener de su propio contenido, también una cabecera (header) y posiblemente un pie (footer).


<aside>

La etiqueta <aside> se usa para marcar un trozo de contenido que está tangencialmente relacionado con el contenido que lo rodea, pero que no es parte del mismo. Ejemplos de uso serían: glosario de términos, grupos de enlaces a páginas relacionadas, barras laterales, …


<nav>

La etiqueta <nav> se usa para marcar la navegación de la página web. Esta etiqueta no debe ser usada para marcar todos los grupos de enlaces, únicamente los bloques principales de navegación por la página.


<footer>

La etiqueta <footer> se usa para marcar el pie de una sección o documento y que contiene información sobre el mismo como el autor, licencia, términos de uso, enlaces a contenido relacionado, avisos legales, etc.


<adress>

El elemento <adress> representa la información del contacto del artículo o elemento padre. Si dicho elemento padre es el body, entonces se refiere al contacto de información de todo el documento.
